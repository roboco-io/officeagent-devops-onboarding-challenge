# 발표 자료 (`deck/`)

본 디렉토리에 30분 발표 + 30분 Q&A용 자료를 작성하세요.

## 형식 (자유)

- PDF (가장 무난)
- Keynote / PowerPoint 파일
- Google Slides 공유 링크 + PDF export
- Markdown deck (예: marp, slidev) — 빌드 결과 PDF 또는 정적 페이지 포함

## 분량 가이드

- 30분 발표 ≈ 슬라이드 15~25장
- 시작 1~2장: 본인이 깊이 다룬 도메인 2개 + 가장 큰 미해결 위험 명시
- 본론: 핵심 의사결정 3~5개 (각 1~3장)
- 마무리 1~2장: 검증 계획 · 후속 과제

## 평가 관점

- **자료 구조·명료성** (5점): 의사결정 3~5개가 트레이드오프 표로 정리되었는가
- **Q&A 응답 깊이** (7점): 의사결정 근거를 흔들림 없이 설명, 모르면 솔직히 인정
- **미해결 위험 솔직성** (3점): 가장 큰 위험 1~2개를 스스로 명시 (감점 아닌 가산)

## 표준 시작 질문 (평가자 공통)

발표 시작 5분 안에 평가자가 묻습니다:

> "본인이 깊이 다룬 2개 도메인 중, 가장 큰 미해결 위험이라고 본 항목 1개와 그렇게 본 근거를 먼저 말씀해 주세요."

본 질문에 첫 자리에서 답할 수 있도록 자료를 구성하시면 좋습니다.

## Q&A 5축

평가자가 Q&A 30분 안에 다음 5축으로 질문합니다 (각 5~7분):

1. 의사결정 근거 — "X 채택 이유, 다른 대안 Y의 단점"
2. 채택하지 않은 대안의 단점 인정 — "본인 설계가 깨질 시나리오 1개"
3. 1차 자료 출처 — "이 결정 근거가 된 docs/가이드의 어느 페이지·문장"
4. NHN-only 종속 식별 — "오픈스택 단계에서 재작성 불가피한 부분"
5. 학습 궤적 — "retros 중 가장 막혔던 지점과 돌파 방법"

자료 안에 위 5축에 답할 수 있는 참고 슬라이드(부록)를 미리 준비해 두면 좋습니다.

## 제출 시점

마감일 기준 리포에 발표 자료가 포함되어 있어야 합니다. **당일 슬라이드 추가 제출 불가.**
