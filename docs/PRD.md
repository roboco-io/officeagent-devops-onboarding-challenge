# OfficeAgent DevOps/AIOps 채용 과제

> 주니어 DevOps/AIOps (정규직). 첫 프로젝트: **AWS → NHN 클라우드 → (장기) 오픈스택 마이그레이션 설계**.
> 회사의 모든 개발자는 **바이브 코딩 기반 풀스택+풀사이클 개발자**를 지향합니다. 이 채용은 그 첫걸음을 함께할 사람을 찾는 자리입니다.

## 채용 개요

NextIntelligence는 OfficeAgent 인프라를 **AWS → NHN 클라우드 → (장기) 오픈스택 기반 온프레미스**로 단계적 이전하려 합니다. 이 마이그레이션을 첫 프로젝트로 맡으면서, 장기적으로 회사의 다른 모든 개발자처럼 **바이브 코딩 기반 풀스택+풀사이클 개발자**로 성장할 정규직 주니어를 채용합니다.

- **고용 형태**: 정규직 (FTE)
- **시니어리티**: 신입 ~ 1년 (부트캠프 수료자·전환자 환영)
- **멘토링**: 30년+ AWS 경력 시니어 엔지니어가 코칭
- **채용 채널**: 회사가 멘토링하는 인력풀 내 추천·직접 컨택
- **제출 마감**: **2026-05-31 (일) 23:59 KST** (리서치·설계·발표 자료 준비 포함)

## 과제 본질

별첨 [`AWS_BASELINE.md`](#부록-별첨-aws-스펙-골격)의 OfficeAgent AWS 구성도를 출발점으로, **NHN과 (장기) 오픈스택 둘 다에 대응 가능한 마이그레이션 설계**를 작성하고 발표 세션에서 의사결정·트레이드오프를 설명하세요.

**주니어 채용입니다.** 따라서 평가는 "정답을 아는가"가 아니라 "**낯선 요건(NHN·CSAP·오픈스택)을 어떻게 공부·조사·분해해서 적합한 계획을 세우는가**"에 집중합니다. 코드 구현은 선택이며, 부분 구현·미완 영역도 의사결정 흔적이 명확하면 가산점입니다.

핵심 원칙:
- **핵심 결정에는 잠정 결론을 내세요** — 미확인 영역은 "후속 조사 필요"로 넘기지 말고 **리스크 + 검증 계획**을 함께 적습니다. 단순 미답변과 정직한 리스크 관리를 구분합니다.
- **넓고 얕게보다 좁고 깊게** — 네트워크·데이터·운영관측 등 여러 도메인을 모두 다루기보다, **2개를 정해 깊이 파는** 편이 더 높이 평가됩니다.

## 1. 필수 제출물

### 1.1 `MIGRATION_PLAN.md` — 마이그레이션 설계서

다음 5개 도메인 중 **2개 이상을 "깊은 분석" 수준**으로 다루세요. 나머지는 개요·매핑·미해결 가정 정도만 적어도 됩니다.

| 도메인 | 깊은 분석 시 기대 |
|--------|------------------|
| **A. 네트워크 / 보안** | VPC ↔ NHN VPC 대응, 망분리 zone, IAM/STS ↔ NHN 권한, CSAP 인증 환경 요건 반영 |
| **B. 데이터 / 스토리지** | RDS↔NHN DB, S3↔Object Storage, 데이터 이전 단계·다운타임·일관성 검증 |
| **C. 컴퓨트 / 배포** | ECS·Fargate ↔ NHN Kubernetes 또는 VM, 배포 파이프라인, 블루/그린·롤백 |
| **D. 운영·관측 / AIOps** | CloudWatch ↔ NHN 모니터링·로그, 알람 진단 자동화, AIOps 통합 지점 |
| **E. 비용 / 규제** | 인스턴스·트래픽·라이선스 비용 비교, CSAP·데이터 주권·계열사 정책 영향 |

공통으로 모든 산출물에 포함되어야 할 것:
- AWS → NHN → 오픈스택 **3단계 흐름의 큰 그림** (요약 한 페이지)
- 각 단계의 **롤백/단축 경로**
- 본인이 "깊이 다룬 도메인"에서 **잠정 결론** 명시
- **공식 1차 자료 인용** (NHN docs · CSAP 가이드 · 오픈스택 docs URL/캡처)

### 1.2 `ARCHITECTURE.md` — 추상화 설계

같은 애플리케이션을 NHN과 (장기) 오픈스택 둘 다 배포 가능한 구조를 어떻게 분리할지 **다이어그램 + 짧은 본문**으로 설명. 코드 수준 IaC 추상화는 **선택**(§2)이며, 본 문서는 **문서·다이어그램 수준**의 분리 설계로 충분합니다.

- 다이어그램 (mermaid / draw.io / Excalidraw / 손그림 스캔 자유)
- NHN-only / 공통 / 오픈스택-only 영역 구분
- 채택 도구·서비스의 의사결정 흔적: 왜 이걸 골랐는가, 다른 대안의 단점

### 1.3 발표 자료 (`deck/` 또는 단일 PDF)

- **30분 발표 + 30분 Q&A** 가정
- 핵심 의사결정 3~5개를 **트레이드오프 표**로 요약
- 가장 큰 미해결 위험 1~2개를 솔직히 명시 (인정해도 감점 없음, 오히려 가산)
- 마감 시점에 리포에 포함 (당일 추가 제출 불가)

### 1.4 `retros/` — 회고 / 학습 일지

`challenge-template/` 의 Retrobot이 push 시점마다 KPT 회고를 자동 생성합니다. 본인이 AI(Claude Code SDK 또는 Codex CLI)에 입력한 프롬프트 · 세션 · 시행착오가 추출되어 **학습 궤적의 1순위 사료**가 됩니다.

**Retrobot 또는 유료 AI 구독을 사용하지 않는 경우** 동등한 학습 일지(`retros/*.md`)를 직접 작성해 제출하세요. **평가상 불이익 없습니다.** (§기술 조건 참조)

## 2. 선택 제출물 (가산점, 부분 구현 환영)

가산 폭은 평가 비중 안에서 흡수됩니다 (별도 가산 점수 없음). 구현 성향이 강하면 도전, 아니면 안 해도 본 평가에서 불리하지 않습니다.

- **IaC PoC** (`infra/`): NHN/오픈스택 둘 다 적용 가능한 모듈 일부. 도구 자유(Terraform / OpenTofu / Pulumi / Crossplane / Ansible 등). `plan` / `preview` 산출물도 가산.
- **운영 자동화 PoC** (`runbooks/`, `scripts/`): 알람 진단 · 드리프트 비교 · 비용 이상치 감지 등 AIOps 시나리오에 LLM SDK를 통합.
- **검증·관측 설계**: 마이그레이션 전후 동일성 검증 방법론, 메트릭/로그 대시보드 정의.

**Q. NHN 계정이 필요한가요?**
**불필요**. 실 apply 없음. 설계 + (선택) `plan`/`dry-run` 산출물로 충분.

**Q. IaC 도구를 모르는데 시작해도 되나요?**
가능합니다. **마감일(2026-05-31)까지 어떻게 학습·조사하는지** 자체가 평가 대상입니다. retros에 학습 궤적을 남기세요. IaC PoC는 **선택**이며 안 해도 평가상 불이익 없습니다.

**Q. AI 코딩 도구가 필수인가요?**
권장이지만 **필수 아님**. 유료 구독이 없는 경우 수동 `retros/*.md` 작성으로 대체할 수 있고, 평가 불이익 없습니다.

**Q. 코드 구현은 꼭 해야 하나요?**
**아니오**. 설계 + 발표로 충분. IaC/AIOps PoC가 있으면 의사결정 흔적의 추가 증거로 평가에 반영.

**Q. 5개 도메인을 다 다뤄야 하나요?**
**아니오**. 2개를 정해 "깊은 분석" 수준으로 다루고, 나머지는 매핑·미해결 가정 수준이면 충분합니다. **넓고 얕게보다 좁고 깊게**가 더 높은 평가를 받습니다.

**Q. "모른다"고 적으면 감점인가요?**
미확인 자체는 감점이 아닙니다. 단, **핵심 결정**(본인이 깊이 다루겠다고 선택한 도메인)에는 잠정 결론이 필요합니다. 미확인이라면 리스크 + 검증 계획을 함께 적어 주세요. 단순 미답은 감점입니다.

**Q. 발표 자료 형식은?**
PDF·Keynote·Google Slides·Markdown deck 자유. 마감 시점에 리포에 포함되어 있어야 합니다.

## 부록: 별첨 AWS 스펙 골격

OfficeAgent의 운영 인프라(익명화 버전)는 다음 수준입니다. 후보는 이 골격을 출발점으로 마이그레이션 설계를 진행합니다. **실제 운영 인프라 전체가 아니라 후보 평가 범위로 한정된 가상 스펙입니다.**

```
[AWS 리전: ap-northeast-2]
│
├── Network
│   ├── VPC (10.0.0.0/16)
│   ├── Public subnet × 2 (ALB·NAT Gateway)
│   ├── Private subnet × 2 (앱·DB)
│   └── Security Groups (ALB → App → DB 흐름 제한)
│
├── Compute
│   ├── Application Load Balancer (ALB)
│   ├── ECS Fargate Service × 1 (FastAPI 백엔드, 2 task)
│   └── ECR (컨테이너 레지스트리)
│
├── Data
│   ├── RDS PostgreSQL (Multi-AZ, db.t3.medium)
│   ├── S3: 문서 업로드 버킷 + 정적 자산 버킷
│   └── ElastiCache Redis (캐시·세션, single node)
│
├── Identity & Secrets
│   ├── IAM Role (ECS Task Role / Execution Role)
│   └── AWS Secrets Manager (DB credential, API key)
│
├── Observability
│   ├── CloudWatch Logs (앱·ALB 액세스)
│   ├── CloudWatch Metrics + Alarms (CPU·error rate)
│   └── X-Ray (선택, 일부 트레이스)
│
├── CI/CD
│   ├── GitHub Actions (build + ECR push + ECS deploy)
│   └── Trivy 이미지 스캔
│
└── External
    ├── Anthropic API · OpenAI API (LLM 호출)
    └── 외부 SaaS (Slack notification)
```

**가상 트래픽 규모**: 동시 사용자 < 200, 일일 RPS 평균 5 / 피크 50, 문서 업로드 일평균 500건.

**가상 컴플라이언스 요건**: 향후 공공 부문 도입을 위해 CSAP 인증 환경 대응 필요. 망분리·암호화·감사 로그 요건이 NHN 단계부터 적용된다고 가정.

**가상 운영 제약**: 마이그레이션 동안 다운타임 허용 시간 **최대 4시간** (주말 야간 1회). 데이터 이전 무손실 필수.
