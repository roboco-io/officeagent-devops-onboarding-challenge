# 망분리 — 물리적/논리적

[[csap|CSAP]] 등급을 가르는 핵심 기술 요건. OfficeAgent 멀티 환경 배포 설계에서 목표 등급에 따라 망분리 방식이 결정된다.

## 두 가지 방식

| 구분 | 물리적 망분리 | 논리적 망분리 |
|------|---------------|---------------|
| 정의 | 완전히 독립된 네트워크 인프라(서버·장비 분리) | VPC/VLAN/SDN 기반, 물리 장비 공유 + 논리 분리 |
| 적용 | 민감 개인정보·국가 중요 정보·국방·외교 | 일반 공공 업무, 비중요 정보 |
| [[csap]] 등급 | 상등급 | 중·하등급 허용 |
| 클라우드 적용 | 곤란 (인프라 비용·관리 복잡) | 퍼블릭 클라우드 적용 가능 |
| 비용 | 높음 | 낮음 |

## 법적 근거

- 정보통신망법 제25조 (안전성 확보 조치)
- 개인정보보호법 제29조 (개인정보 안전조치 의무)

## 클라우드 환경에서의 논리적 망분리

- 서버 가상화 + 네트워크 가상화 기술 기반, 논리적으로 분리된 가상 네트워크상에 구성
- [[csap]] 하등급은 논리적 망분리만으로 취득 가능 → 국내 SaaS 사업자 신규 진입 허용, [[foreign-csp|외산 CSP]]도 노리는 경로
- 단, 클라우드 시스템·백업·데이터의 **물리적 위치는 국내로 한정** ([[data-sovereignty|데이터 주권]] 참조)
- [[nhn-cloud|NHN Cloud]]는 공공기관 전용 리전을 민간과 논리적으로 분리 운영

## 2024+ 동향

하이브리드 시나리오를 고려한 망분리 차등화·유연화. 공공기관 업무 특성·데이터 중요도에 따라 망분리 수준 차등 적용 ([[policy-trends]] 참조).

## OfficeAgent 마이그레이션 설계 시사점

- AWS의 VPC·Subnet·Security Group 구조가 NHN VPC / 오픈스택 Neutron에서 어떻게 대응되는지 매핑 필요
- 목표 [[csap]] 등급이 상등급이면 물리적 망분리 → 퍼블릭 클라우드만으로는 불가, [[on-premise|오픈스택 온프레미스]] 검토
- AWS NAT Gateway 등가 서비스를 NHN에서 확인
- [[migration-checklist]] 망분리 항목 참조
