# [RAW] 한국 공공·정부기관 클라우드 도입 규제 — 종합 조사

> 출처: Perplexity Deep Research (Sonar), 조사일 2026-05-20. **불변 원본** — 수정 금지, 갱신 시 새 파일로 드롭.
> 용도: `wiki/` 페이지 컴파일 소스.

## CSAP 인증 — 제도 기반

- 법적 근거: 「클라우드컴퓨팅 발전 및 이용자 보호에 관한 법률」 제23조의2 [출처 law.go.kr]
- 정책기관: 과학기술정보통신부 / 인증기관: 한국인터넷진흥원(KISA) / 평가기관: KISA·지정기관 / 공공부문 기술자문기관: 국가보안기술연구소 / 인증위원회
- 운영 프로세스: 인증 신청 → 서면 평가 → 현장 평가 → 취약점 점검 → 인증 심사 → 인증서 발급
- 목적: 국가·공공기관에 안전성·신뢰성이 검증된 민간 클라우드 공급
- 법적 강제성: 공공기관 핵심 행정 정보 시스템은 사실상 CSAP 인증 서비스만 도입 가능 (행안부 가이드·과기정통부 고시로 간접 강제)

## CSAP 등급제 (2023-12 개편)

- 2023년 12월 국가기관 시스템 중요도 분류 기준에 따라 상·중·하 3단계 도입
- 2024년 2월 「클라우드컴퓨팅서비스 보안인증에 관한 고시」 일부개정(안) 행정예고
- 기존 IaaS 표준 / SaaS 표준·간편 / DaaS 체계 → 상·중·하로 전환
- 통제항목 수:
  - IaaS: 14개 분야 116개 통제항목
  - SaaS 표준등급: 13개 분야 79개 / SaaS 간편등급: 11개 분야 31개
  - 개정안: SaaS 상등급 14개 분야 64개 / SaaS 하등급 11개 분야 30개
  - DaaS: 14개 분야 110개 통제항목
- 상등급 = 중등급 + 추가 4개 평가항목 (물리적 망분리·고급 암호화·실시간 모니터링·강화 접근통제 추정)
- 하등급: 논리적 망분리만으로 취득 가능 → AWS 등 외산 CSP가 노리는 진입점
- 유효기간: 5년. 최초 평가 후 1년 경과 시 사후평가, 이후 매년 1회(총 4회) + 만료 6개월 전부터 갱신 신청
- 국제 대응: 미국 FedRAMP HIGH/MOD/LOW ↔ CSAP 상/중/하 (LI-SaaS ↔ 하등급). 직접 상호 인증은 없음
- 취득 비용·기간: 6개월~1년, 5천만원~1억원+ (인증 수수료 + 보안 체계 구축 + 컨설팅 + 인프라 개선)

## 주요 정부 기관 역할

- 과학기술정보통신부: 클라우드컴퓨팅법 §5(기본계획), §23조의2(보안인증). CSAP 정책·고시
- 한국인터넷진흥원(KISA): CSAP 인증 신청·심사·발급, 사후·갱신 평가
- 행정안전부: 전자정부법 §35조의2. 공공기관 정보화법 시행령(2023-12 개정)으로 인증 서비스 우선 권고
- 국정원: 국가보안법·정보통신망법. 국가 중요 시스템(국방·외교) 보안 감독, 물리적 망분리 요구
- 국가보안기술연구소: CSAP 공공부문 기술자문
- 개인정보보호위원회: 개인정보보호법 §24·§29

## 망분리

- 물리적 망분리: 완전 독립 네트워크 인프라. 민감 개인정보·국가 중요 정보·국방·외교 시스템 (CSAP 상등급)
- 논리적 망분리: VPC/VLAN/SDN 기반. CSAP 중·하등급 허용 (퍼블릭 클라우드 적용 가능)
- 근거: 정보통신망법 §25, 개인정보보호법 §29
- 2024+ 동향: 하이브리드 시나리오 고려한 차등화·유연화

## 데이터 주권

- 클라우드컴퓨팅법 §18: 근거지(데이터 처리 위치) 선택 원칙 — 공공기관 요청 시 국내 한정
- 개인정보보호법 §24·§28의2: 개인정보 국외 이전 제한
- 정보통신망법 §25: 안전성 확보 조치
- 실무: 데이터 저장·처리 국내 리전 의무, KMS 독립 운영, 데이터 처리 계약서에 데이터 주권 명시 의무화(2024+)

## 추가 인증

- ISMS / ISMS-P (정보보호 / 정보보호+개인정보보호 관리체계): KISA. CSAP과 별개
- K-ISMS: 한국 표준 정보보호 인증
- ISO/IEC 27001: 국제 표준. CSAP과 일부 통제항목 중복으로 우대 가능

## 외산 CSP (AWS·Azure·GCP) 제약

- 데이터 주권·물리적 망분리·국내 운영 요건으로 상·중등급 직접 취득 사실상 불가
- 하등급(논리적 망분리)은 일부 진입 시도. AWS 서울 리전 기반
- 2024-12 이후 CSAP 대개편 논의에서 외산 CSP 진입 완화가 쟁점

## 온프레미스(오픈스택) 고려 이유

- 국방·국가 중요 시스템: CSAP 상등급 + 물리적 망분리 절대 요구
- 데이터 주권 절대 요구 분야 (외교·국방·정보)
- CSP 의존성(가격·정책 리스크) 제거
- 자체 구축 시 ISMS-P·K-ISMS 등 별도 인증 부담 → 비용·시간 트레이드오프 큼

## 2025-2026 정책 동향

- CSAP 대개편 논의 2024-12 본격화
- 외부 전문기관 취약점 점검 허용 (2024+)
- 동일 서비스 2개 이상 등급 신청 시 중복 평가항목 생략
- 하이브리드·멀티 클라우드 가이드라인 행안부 차원 표준화
